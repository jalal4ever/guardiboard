# Instructions pour lire la documentation
Utilise cette compétence lorsque tu as besoin de comprendre les décisions d'architecture (ADR), le fonctionnement des API internes ou la topologie de la base de données.

1. Liste les fichiers présents dans le dossier contenant la documentation de ce projet (ex: `/docs`).
2. Lis attentivement le contenu du fichier Markdown correspondant à ton besoin technique actuel.
3. Applique strictement les recommandations architecturales et les conventions trouvées dans ces documents à ton code généré.
